# 需求文档

## 1. 项目概述

三国杀TCG抽卡模拟器，用于辅助线下实体卡牌测试。模拟两名选手从各自军营（预组）中随机抽取武将上阵的对局过程。

## 2. 核心需求

### 2.1 预组与军营

- 每位选手携带1个预组，预组即"军营"，包含39名武将卡
- 双方仅能从自己的军营中获取武将
- 当前提供4种预组，双方可自由选择（允许相同预组）：

### 2.2 抽卡规则

- 每回合上阵2名武将（从军营中随机抽出2张武将卡）
- **先手玩家的首个回合仅上阵1名武将**
- 已上阵的武将不可再次上阵（从军营中移除）
- 军营中武将数不足以上阵指定数量时，提示"武将数不足，本次模拟结束"
- **战场上限校验**：抽卡前校验先手和后手两方战场武将均≤5才允许抽卡；若任一方战场存在“徐庶”，则无需任何校验即可抽卡

### 2.3 回合流程

1. 先手玩家首回合：上阵1名武将
2. 后手玩家回合：上阵2名武将
3. 先手玩家回合：上阵2名武将
4. 后手玩家回合：上阵2名武将
5. 重复3-4，直到有一方军营不足或用户关闭页面

### 2.4 主帅卡

- 选择预组后，对应的主帅卡自动出现在战场上（例如选"蜀国-主帅刘备"则战场出现一张"刘备"）
- 主帅卡带有♛标记，与普通武将卡区分
- 主帅卡id格式: `{deckId}_commander_{commanderName}`，isCommander=true

### 2.5 休整区

- 每名选手有独立的休整区
- 战场上的武将卡可置入休整区（点击卡牌右上角"x"按钮）
- 进入休整区时，乐不思蜀状态自动消失
- 休整区以"叠卡"形式显示（最上方为最近置入的卡），随卡牌增多看起来更厚
- 点击休整区"叠卡"时弹窗显示所有休整区武将，悬浮时显示"恢复至战场"按钮
- 点击"恢复至战场"可将武将卡从休整区重新放回战场

### 2.6 乐不思蜀

- 战场上的武将卡可标记乐不思蜀状态（点击卡牌上方"乐"按钮切换）
- 标记后武将卡上方显示"乐不思蜀"浮动标签
- 进入休整区时乐不思蜀状态自动消失

### 2.7 撤回功能

- 新增"撤回上一次抽卡"按钮
- 点击后撤回上一次抽卡操作：将抽出的武将放回军营
- 撤回后再点击"下一步"时，会抽回同一批武将（非重新随机抽取）
- 每次抽卡后只能撤回一次，撤回后按钮禁用
- **撤回时放回位置**：如果上次抽卡中有来自军营顶部的武将，撤回时这些武将放回军营顶部；其他武将放回军营

### 2.8 武将技能弹窗

- 点击战场/休整区中的武将卡时，弹出技能信息弹窗
- 弹窗包含：武将名称、主帅标记（如有）、技能图片（预留口子）、技能描述（预留口子）
- 技能描述和图片路径后续填充，当前显示"技能描述待补充"

### 2.9 操作方式

- 用户通过"下一步"按钮逐步推进，每次点击执行一个上阵动作
- 模拟过程不自动推进，必须用户手动触发每一步
- "撤回上一次抽卡"按钮可撤回最近一次抽卡操作
- 武将卡可交互：置入休整区/恢复至战场/切换乐不思蜀/放回军营顶部/查看技能弹窗
- **约束**：点击放回军营顶后，撤回上一次抽卡按钮被禁用

### 2.10 放回军营顶部

- 战场上的武将卡可放回军营顶部（点击卡牌悬浮"↑"按钮）
- 放回军营顶部时，乐不思蜀状态自动消失
- 军营顶部为LIFO栈（后进先出）：最后放回的武将最先被抽出
- 示例：战场有A、B、C、D、E，依次将B、C、E放回军营顶 → 下次抽2张时抽出E、C；再轮到我抽卡时抽出B和另一张未知武将
- **约束**：点击放回军营顶后，撤回上一次抽卡按钮被禁用（防止状态冲突）

### 2.11 自定义抽卡数量

- "下一步"按钮旁显示抽卡数量输入框（1-4），默认为2
- 先手首回合默认显示1
- 用户可修改抽卡数量（1-4），点击"下一步"时按指定数量抽卡
- 每次抽卡完成后，抽卡数量自动重置为2

## 3. 前端需求

### 3.1 页面布局

- 上方区域：先手玩家
- 下方区域：后手玩家
- 中间：控制按钮与状态信息

### 3.2 交互需求

- 双方各有一个下拉框，用于选择预组
- 双方预组均选定后，"开启模拟"按钮可用，否则需校验提示
- 开启模拟后：
  - 下拉框禁用，防止更改
  - "下一步"按钮可用，旁边显示抽卡数量输入框（默认先手首回合1，其余2）
  - "撤回上一次抽卡"按钮可用（仅在抽卡后）
  - "重置"按钮可用
  - 主帅卡自动出现在战场
- 点击"下一步"执行当前行动方的上阵动作，按输入框中的数量抽卡
- 每次抽卡后抽卡数量输入框自动重置为2
- 武将不足时禁用"下一步"，提示结束
- 战场武将>5时阻止抽卡（双方均校验），提示弃置；若存在徐庶则豁免
- "重置"清空所有状态回到初始

### 3.3 视觉需求

- 三国古风主题，花里胡哨的好看
- 势力颜色区分：魏(蓝)、蜀(红)、吴(绿)、群(紫)
- 卡牌上阵动画效果
- 当前行动方高亮标识
- 军营剩余数量与进度条显示
- 战斗日志记录操作历史
- 休整区叠卡效果（卡牌堆积视觉）
- 乐不思蜀浮动标签动画
- 主帅卡♛标记
- 弹窗（休整区详情/技能信息）
- 卡牌悬浮操作按钮（"x"置入休整区+"乐"乐不思蜀+"↑"放回军营顶部）
- 抽卡数量输入框（1-4，每次抽卡后重置为2）

### 3.4 图片需求

- 前端代码预留图片替换口子，仅需提供图片路径即可更换
- 图片加载失败时有文字占位容错
- 技能描述和技能图片预留口子（SkillConfig配置类）

## 4. 技术约束

- 无数据库，全部数据在内存中（会话级）
- 轻量级，可在任何电脑上使用（Java 8+环境即可）
- Spring Boot内嵌Tomcat，打为可执行JAR

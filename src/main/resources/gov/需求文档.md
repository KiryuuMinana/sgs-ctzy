# 需求文档

## 1. 项目概述

三国杀TCG抽卡模拟器，用于辅助线下实体卡牌测试。模拟两名选手从各自军营（预组）中随机抽取武将上阵的对局过程。

## 2. 核心需求

### 2.1 预组与军营

- 每位选手携带1个预组，预组即"军营"，包含39名武将卡
- 双方仅能从自己的军营中获取武将
- 当前提供4种预组，双方可自由选择（允许相同预组）：

### 2.2 抽卡规则

- 每回合上阵2名武将（从军营中随机抽出2张武将卡）
- **先手玩家的首个回合仅上阵1名武将**
- 已上阵的武将不可再次上阵（从军营中移除）
- 军营中武将数不足以上阵指定数量时，提示"武将数不足，本次模拟结束"

### 2.3 回合流程

1. 先手玩家首回合：上阵1名武将
2. 后手玩家回合：上阵2名武将
3. 先手玩家回合：上阵2名武将
4. 后手玩家回合：上阵2名武将
5. 重复3-4，直到有一方军营不足或用户关闭页面

### 2.4 操作方式

- 用户通过"下一步"按钮逐步推进，每次点击执行一个上阵动作
- 模拟过程不自动推进，必须用户手动触发每一步

## 3. 前端需求

### 3.1 页面布局

- 上方区域：先手玩家
- 下方区域：后手玩家
- 中间：控制按钮与状态信息

### 3.2 交互需求

- 双方各有一个下拉框，用于选择预组
- 双方预组均选定后，"开启模拟"按钮可用，否则需校验提示
- 开启模拟后：
  - 下拉框禁用，防止更改
  - "下一步"按钮可用
  - "重置"按钮可用
- 点击"下一步"执行当前行动方的上阵动作
- 武将不足时禁用"下一步"，提示结束
- "重置"清空所有状态回到初始

### 3.3 视觉需求

- 三国古风主题，花里胡哨的好看
- 势力颜色区分：魏(蓝)、蜀(红)、吴(绿)、群(紫)
- 卡牌上阵动画效果
- 当前行动方高亮标识
- 军营剩余数量与进度条显示
- 战斗日志记录操作历史

### 3.4 图片需求

- 前端代码预留图片替换口子，仅需提供图片路径即可更换
- 图片加载失败时有文字占位容错

## 4. 技术约束

- 无数据库，全部数据在内存中（会话级）
- 轻量级，可在任何电脑上使用（Java 8+环境即可）
- Spring Boot内嵌Tomcat，打为可执行JAR

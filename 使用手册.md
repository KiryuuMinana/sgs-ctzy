# 三国杀TCG - 抽卡模拟器 使用手册

## 一、这是什么？

这是一个 **三国杀TCG（集换式卡牌）抽卡模拟器**，用于辅助线下实体卡牌测试。

它可以模拟两名选手从各自军营（预组牌堆）中随机抽取武将上阵的完整对局过程，帮助你在线上进行抽卡模拟，无需实体卡牌。

---

## 二、运行前的准备 —— 安装 Java

本程序需要 **Java 运行环境（JRE/JDK 8 或以上版本）** 才能运行。如果你的电脑还没有安装 Java，请按以下步骤操作：

### 步骤 1：下载 Java

1. 打开浏览器，访问以下任一地址下载 Java：
   - **推荐（Eclipse Temurin）**：https://adoptium.net/
     - 进入后点击下载按钮，选择 **JDK 8** 或 **JDK 17** 版本
     - 操作系统选择 **Windows**，架构选择 **x64**
     - 下载 `.msi` 安装包
   - **Oracle 官方**：https://www.oracle.com/java/technologies/downloads/

2. 下载完成后，双击安装包开始安装

### 步骤 2：安装 Java

1. 双击下载的安装文件（如 `OpenJDK8U-jdk_x64_windows_hotspot_8u412b08.msi`）
2. **安装过程中，请勾选"Set JAVA_HOME variable"（设置环境变量）选项**（非常重要！）
3. 一路点击 **"Next"（下一步）** → **"Install"（安装）** → **"Finish"（完成）**
4. 安装完成后，**重启电脑**（确保环境变量生效）

### 步骤 3：验证 Java 是否安装成功

1. 按键盘 `Win + R`，输入 `cmd`，回车，打开命令提示符窗口
2. 在黑色窗口中输入以下命令并回车：
   ```
   java -version
   ```
3. 如果看到类似以下内容，说明安装成功：
   ```
   openjdk version "1.8.0_412"
   OpenJDK Runtime Environment (Temurin)(build 1.8.0_412-b08)
   OpenJDK 64-Bit Server VM (Temurin)(build 25.412-b08, mixed mode)
   ```
4. 如果提示"不是内部或外部命令"，说明环境变量未生效，请重启电脑后重试

## 三、启动程序

你需要以下两个文件（它们应该放在同一个文件夹中）：

| 文件名 | 说明 |
|--------|------|
| `sgs-ctzy-demo.jar` | 程序主体文件（约29MB） |
| `启动抽卡模拟器.bat` | 一键启动脚本 |

### 启动方法

1. **双击** `启动抽卡模拟器.bat` 文件
2. 会弹出一个黑色命令行窗口，显示启动过程
3. 当看到类似以下内容时，说明启动成功：
   ```
   Started SgsCtzyApplication in 0.827 seconds
   ```
4. 打开浏览器（推荐 Chrome / Edge / Firefox），在地址栏输入：
   ```
   http://localhost:8080
   ```
5. 即可看到抽卡模拟器页面

### 停止程序

- **关闭黑色命令行窗口** 即可停止服务
- 关闭后浏览器页面将无法访问，这是正常的

## 四、功能使用说明

### 4.1 选择预组（牌堆）

页面打开后，你会看到 **上方和下方各有一个下拉框**，分别代表两位玩家的牌堆选择。

当前提供 **4种预组牌堆**，双方可以自由选择（允许选相同的）：

| 势力 | 主帅 | 说明 |
|------|------|------|
| 魏国 | 曹操 | 蓝色主题 |
| 蜀国 | 刘备 | 红色主题 |
| 吴国 | 孙尚香 | 绿色主题 |
| 群雄 | 貂蝉 | 紫色主题 |

每个预组包含 **39张武将卡**。

**操作**：分别在两个下拉框中选择预组，两个都选好之后，**"开启模拟"** 按钮会亮起。

### 4.2 开启模拟

1. 点击 **"开启模拟"** 按钮
2. 双方的主帅卡会 **自动出现在战场上**（例如选蜀国就会出现"刘备"）
3. 主帅卡带有 ♛ 皇冠标记，与普通武将区分
4. 下拉框会被锁定，不能再更改

### 4.3 抽卡上阵（下一步）

点击 **"下一步"** 按钮，当前行动方会从军营中随机抽取武将上阵。

**抽卡规则**：
- 先手玩家的 **第一个回合** 只抽 **1张**
- 之后每回合抽 **2张**
- 先手抽完轮到后手，轮流进行

**可以调整的参数**：
- **抽卡数量**："下一步"旁边的输入框，可设置 1-4 张（每次抽卡后自动恢复为2）
- **抽卡顺序**：旁边的下拉框可选择"先手抽卡"或"后手抽卡"，默认轮流（每次抽卡后自动恢复为轮流）

### 4.4 武将卡操作

将鼠标 **悬浮** 在战场上的武将卡上时，卡牌上方会出现一排操作按钮：

| 按钮 | 名称 | 功能说明 |
|------|------|----------|
| **×** | 置入休整区 | 将武将从战场移入休整区（休息中），同时弹出该武将图片 |
| **乐** | 乐不思蜀 | 给武将贴上"乐不思蜀"标记（浮动标签），再点一次取消 |
| **↑** | 放回军营顶 | 将武将放回军营最上方，下次抽卡时优先抽出 |
| **↓** | 放回军营底 | 将武将放回军营最下方，最后才会被抽出 |
| **↻** | 横置 | 将武将卡旋转90°横放，再点一次恢复竖放（仅视觉标记） |

**点击武将卡本身**（不是按钮）：弹出该武将的技能信息弹窗（技能描述和图片预留中）。

### 4.5 休整区

- 每位玩家有独立的休整区，显示为 **叠卡** 形式（卡牌堆叠效果）
- **点击休整区的叠卡**：弹出窗口显示所有休整区武将
- 悬浮某张休整区武将 → 显示 **"恢复至战场"** 按钮
- 点击"恢复至战场"→ 武将从休整区回到战场

**特别注意**：武将进入休整区时，身上的"乐不思蜀"标记会自动消失。

### 4.6 撤回上一次抽卡

- 点击 **"撤回上一次抽卡"** 按钮，可以撤销上一次的上阵操作
- 撤回后，武将会被放回军营中（放回原来的位置）
- 撤回后再次点击"下一步"，会 **抽出同一批武将**（不是重新随机）
- 每次抽卡后只能撤回一次
- **注意**：如果把武将放回军营顶或军营底后，撤回按钮会被禁用

### 4.7 骰子功能（右侧栏）

页面 **右侧** 有一个独立的骰子面板，用于辅助随机数抽取：

1. **调整范围**：下拉框可选择 1-15，代表随机数的最大值（默认为4）
2. **投掷骰子**：点击骰子，骰子会播放旋转动画（1.5秒）
3. **查看结果**：动画结束后，下方的结果展示框会显示抽取的数字
   - 例如：选择4，则从 1、2、3、4 中随机抽一个
4. 骰子功能完全独立，不影响游戏进行

### 4.8 战斗日志

页面中间的 **战斗日志** 区域会记录所有操作历史，包括：
- 上阵（抽出了哪些武将）
- 撤回
- 放回军营顶/底
- 贴/移除乐不思蜀
- 移入休整区 / 恢复至战场

方便回顾整个对局过程。

### 4.9 重置

点击 **"重置"** 按钮可以清空所有状态，回到初始界面，重新开始一局新的模拟。

## 五、常见问题

### Q1：双击 .bat 文件后提示"未检测到 Java"？

**原因**：你的电脑没有安装 Java，或者安装后没有重启电脑。

**解决**：按照上面"二、运行前的准备"安装 Java，安装后重启电脑，再双击 .bat 文件。

### Q2：浏览器访问 http://localhost:8080 显示"无法访问此网站"？

**原因**：程序可能还没启动完成，或者启动失败了。

**解决**：
1. 检查黑色窗口中是否有 `Started SgsCtzyApplication` 字样
2. 如果有错误信息，请截图反馈
3. 确保没有其他程序占用了 8080 端口

### Q3：黑色窗口显示"端口 8080 已被占用"？

**原因**：可能有其他程序（或上一次未关闭的本程序）占用了 8080 端口。

**解决**：
1. 关闭所有黑色命令行窗口
2. 重新双击 .bat 文件启动

### Q4：页面显示正常但点击按钮没反应？

**解决**：
1. 先检查浏览器控制台（按 F12 打开开发者工具 → Console 标签页）是否有红色错误
2. 尝试刷新页面（F5）
3. 如果不行，关闭服务重新启动

### Q5：关闭黑色窗口后，浏览器页面还在但无法操作？

这是正常的。黑色窗口就是服务器，关闭它服务就停了。下次使用需要重新双击 .bat 文件启动。

### Q6：武将放回军营顶/底后，撤回按钮变灰了？

这是设计如此。为了防止状态冲突，放回军营顶/底后不允许撤回上一次抽卡。
